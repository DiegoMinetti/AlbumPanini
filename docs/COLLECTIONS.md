# Authoring collection packages

A collection is a single JSON file in `public/collections/`, referenced by the
manifest `public/collections/index.json`. The app discovers and validates them
at runtime — **no code changes are needed to add a collection.**

## Manifest — `index.json`

```json
{
  "collections": [
    {
      "id": "worldcup-2026",
      "file": "worldcup-2026.json",
      "name": "FIFA World Cup 2026",
      "description": "Official-style sample album.",
      "version": "1.0.0",
      "language": "en"
    }
  ]
}
```

## Package shape

Validated by `collectionPackageSchema` (`src/types/collection.ts`).

```json
{
  "id": "worldcup-2026",
  "schema": 1,
  "name": "FIFA World Cup 2026",
  "description": "…",
  "version": "1.0.0",
  "language": "en",
  "coverImage": "optional/relative/or/url.png",
  "teams": [
    {
      "id": "ARG",
      "name": "Argentina",
      "flag": "🇦🇷",
      "primaryColor": "#75AADB",
      "secondaryColor": "#FFFFFF",
      "order": 0
    }
  ],
  "stickers": [
    {
      "id": "ARG-1",
      "code": "ARG 1",
      "name": "Argentina Player 1",
      "teamId": "ARG",
      "category": "player",
      "type": "regular",
      "rarity": "common",
      "image": "optional.png",
      "order": 1
    }
  ]
}
```

### Field notes

| Field | Notes |
| --- | --- |
| `id` | Unique within the manifest; used as the installed collection id. |
| `schema` | Package schema version (currently `1`). |
| `code` | The **printed** code (e.g. `ARG 1`). Powers OCR + bulk import. It is normalized to `ARG1` internally, so spacing/case/leading zeros are flexible. |
| `teamId` | Optional. Collections without teams (e.g. Pokémon) simply omit it. |
| `category`, `type`, `rarity` | Free-form strings; the UI groups/filters by them. |
| `image` | **Optional.** Player/sticker images are optional assets; the app works fully without them. |
| `*color` | Hex (`#rgb` or `#rrggbb`). |

## Tips

- Codes only need to be unique per collection.
- Use `order` to control display ordering; otherwise array order is used.
- The bundled samples are generated by `scripts/generate-collections.mjs`
  (`npm run collections`) — a good reference for large sets.
- Re-installing/updating a package replaces its catalog rows but **preserves the
  user's inventory**, so you can ship `version` bumps safely.
