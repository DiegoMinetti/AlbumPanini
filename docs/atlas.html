An offline-first PWA for sticker collectors. No backend, no login, no cloud — just IndexedDB, M3 design tokens, and a service worker that actually caches Tesseract. Here is the shape of the code, what every file does, and where it hurts.

I.

The five layers

Pages → Hooks → Services → DB → Stores
01

UI

Pages, components, layout. M3 design tokens via Tailwind.

  • 11 pages · 1 router (hash)
  • ~50 components
  • Material 3 tokens
02

Hooks

Reactive bridge between DB and UI. useLiveQuery everywhere except the manifest.

  • useCollectionData
  • useActiveCollection
  • useTournament
03

Services

Domain logic. Pure modules are unit-testable, DB-touching ones use fake-indexeddb.

  • statsService (pure)
  • filterService (pure)
  • backupService / syncService
04

DB

Dexie schema. Tables namespaced by uid = "${cid}::${localId}". Migrations append-only.

  • 9 tables · v2
  • dbVersionHistory tracked
  • scenarios + matchResults
05

Stores

Zustand with persist + a custom safeStorage (localStorage + in-memory fallback).

  • settingsStore (persisted)
  • uiStore (toasts)
  • scenarioStore · reservationStore
